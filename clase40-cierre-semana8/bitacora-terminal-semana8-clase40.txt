1. ¿Qué aprendí sobre el sistema operativo como base de una plataforma?
Respuesta:
Aprendí que el sistema operativo es la base donde funcionan las aplicaciones y servicios, 
porque administra recursos como memoria, procesos, archivos, usuarios y permisos necesarios 
para que una plataforma opere correctamente.

2. ¿Qué comandos de terminal usé durante la semana?
Respuesta:
Usé comandos como pwd, ls, cd, mkdir, touch, cat, cp, mv, df -h, free -h, uname -a, history, 
ip a, top, entre otros, para navegar, administrar archivos y observar el sistema.

3. ¿Qué aprendí sobre usuarios, grupos y permisos?
Respuesta:
Aprendí que Linux organiza el acceso mediante usuarios y grupos, y que los permisos determinan 
quién puede leer, modificar o ejecutar archivos para mantener la seguridad del sistema.

4. ¿Qué significa mínimo privilegio?
Respuesta:
Significa otorgar a cada usuario o proceso únicamente los permisos necesarios para realizar su función, 
evitando accesos innecesarios que puedan generar riesgos.

5. ¿Por qué un permiso como 777 puede ser riesgoso?
Respuesta:
Porque permite que cualquier usuario tenga permisos de lectura, escritura y ejecución, lo que puede facilitar 
modificaciones no autorizadas o comprometer la seguridad del sistema.

6. ¿Qué aprendí sobre procesos?
Respuesta:
Aprendí que un proceso es un programa en ejecución que utiliza recursos del sistema como CPU y memoria, y 
que puede ser monitoreado para conocer el estado de una aplicación.

7. ¿Qué puede pasar si un proceso se detiene?
Respuesta:
La aplicación o servicio puede dejar de funcionar, generar errores, interrumpir la comunicación con usuarios 
o afectar el funcionamiento de una plataforma.

8. ¿Qué aprendí sobre logs?
Respuesta:
Aprendí que los logs son registros de eventos del sistema que muestran información sobre acciones, 
advertencias y errores ocurridos durante la ejecución de una aplicación.

9. ¿Cómo ayudan los logs al diagnóstico?
Respuesta:
Ayudan a identificar problemas porque permiten revisar qué ocurrió, cuándo ocurrió y obtener 
pistas para encontrar la causa de un error y solucionarlo.

10. ¿Qué aprendí sobre Git y trazabilidad?
Respuesta:
Aprendí que Git permite controlar los cambios de un proyecto mediante commits, ramas y 
registros, facilitando saber qué cambios se hicieron, cuándo y por quién.

11. ¿Qué aprendí sobre archivos .env y tokens?
Respuesta:
Aprendí que los archivos .env sirven para guardar configuraciones sensibles como tokens y 
claves, y que no deben publicarse porque pueden comprometer la seguridad de los servicios.

12. ¿Cómo puede servir Telegram a una comunidad digital?
Respuesta:
Telegram puede funcionar como un canal de comunicación para enviar avisos, recibir reportes, 
compartir información y facilitar la participación de una comunidad.

13. ¿Qué cuidados debo tener antes de integrar Telegram con Node.js o n8n?
Respuesta:
Debo proteger los tokens, usar variables de entorno, evitar publicar credenciales, validar 
la información recibida y configurar correctamente los permisos de acceso.

14. ¿Qué fue lo más importante de la Semana 8?
Respuesta:
Lo más importante fue comprender que una plataforma digital no solo depende del código, 
sino también de la seguridad, administración del sistema, control de accesos, monitoreo, 
trazabilidad y manejo responsable de la información para generar confianza en los usuarios.
