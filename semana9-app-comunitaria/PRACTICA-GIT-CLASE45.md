# Práctica Git - Clase 45

## Objetivo

Practicar ramas, stash, merge fast-forward, merge no-fast-forward y resolución de conflictos.

## Conceptos

-Rama:
Es una línea de trabajo independiente dentro de Git que permite desarrollar cambios sin afectar directamente la rama principal del proyecto.
-Commit:
Es un registro o guardado de los cambios realizados en el repositorio. Permite tener un historial de versiones y regresar a estados anteriores.
-Merge:
Es el proceso de unir los cambios de una rama con otra, normalmente integrando una rama de desarrollo a la rama principal (main).
-Stash:
Es una herramienta de Git que guarda temporalmente cambios no confirmados para poder cambiar de rama o realizar otra tarea sin perder el trabajo actual.
-Fast-forward:
Es un tipo de merge donde Git solo mueve la referencia de la rama principal hacia adelante porque no existen cambios diferentes entre las ramas.
-No-fast-forward:
Es un merge donde Git crea un nuevo commit de unión porque las ramas tienen historiales diferentes y ambas contienen cambios propios.
-Conflicto:
Es una situación donde Git no puede combinar automáticamente los cambios porque dos ramas modificaron la misma parte de un archivo. El desarrollador debe resolver manualmente cuál versión conservar.
